
# なぜTaskelか？

> **タスクを管理するほど、タスクが増えていく。その沼、もう終わりにしませんか？**

---

## あなたの痛み、わかります

### 痛み1: タスク沼

Todoistを入れ直すのは3回目。Notionのデータベースは作ったけど、1週間で開かなくなった。管理すること自体がタスクになっている。

**原因:** 従来のタスク管理ツールは「やることを並べる」だけ。並べれば並べるほど、終わらないリストが罪悪感を生み、ツールを開くのが怖くなる。

### 痛み2: 目標と実行の断絶

年初に「英語を頑張る」「副業を始める」と目標を立てた。3月にはもう忘れている。目標はNotionのどこかに、今日のタスクはTodoistに。2つは繋がっていない。

**原因:** 目標管理と実行管理が別のツール、別の習慣になっている。目標を立てる行為と、今日やることを決める行為が断絶している。

### 痛み3: 時間の見えなさ

「今日やること」を10個並べた。全部やるつもりだった。でも夜になって半分も終わっていない。見積もりが甘いのか、時間が足りないのか、わからない。

**原因:** タスクを「リスト」で捉えている。時間は有限なのに、リストは無限に伸びる。「今日の残り時間」が見えないまま詰め込んでいる。

### 痛み4: 未完了の罪悪感

期限を過ぎたタスクが赤字で並ぶ。見るたびに責められている気がする。結果、ツールを開かなくなる。

**原因:** 従来のツールは未完了を「Overdue」として罰する設計。人間は罰されると逃げる。

---

## Taskelが解決すること

| 痛み | 従来のツール | Taskelの解決策 |
|---|---|---|
| **タスク沼** | やることを並べるだけ | 目標から逆算して「本当にやるべきこと」だけを選ぶ（スコアリング） |
| **目標と実行の断絶** | 目標管理と実行管理が別ツール | 年→月→週→今日が一本の線。双方向に接続 |
| **時間の見えなさ** | リストが無限に伸びる | 終了予定時刻をリアルタイム表示。入りきらないと5秒でわかる |
| **未完了の罪悪感** | Overdueの赤字で罰する | Chute（先送り）は正式な機能。未完了 = 罪にしない |

## 競合との根本的な違い

### Todoist / TickTick / Microsoft To Do

**本質:** やることリスト

リストは作れる。期限も設定できる。でも「それは何のためにやるのか？」「今日本当に終わるのか？」には答えてくれない。目標との接続がなく、時間の有限性も見えない。

| | Todoist等 | Taskel |
|---|---|---|
| 目標との接続 | ❌ なし | ✅ 年→月→週→今日が双方向 |
| 時間の可視化 | ❌ リストのみ | ✅ 終了予定時刻をリアルタイム表示 |
| 未完了の扱い | ⚠️ Overdue（赤字） | ✅ Chute（罰なし） |
| 実行の駆動 | ❌ なし | ✅ ワンクリックタイマー |

### Notion

**本質:** 万能ワークスペース

何でもできる。でも何でもできるがゆえに、タスク管理のための設計がない。データベースを作るところから始まり、運用の設計もユーザー任せ。「管理すること自体がタスク」になりがち。

| | Notion | Taskel |
|---|---|---|
| 目標との接続 | △ 自分で設計すれば可能 | ✅ 最初から構造化されている |
| 実行の駆動 | ❌ なし | ✅ タイマー + セクション + 集中モード |
| 導入コスト | ⚠️ 高い（設計が必要） | ✅ 低い（メソッドに沿って始めるだけ） |

### Motion / Reclaim

**本質:** AIスケジュール最適化

AIがカレンダーの空き枠にタスクを自動配置する。効率は上がるかもしれない。でも「自分で見積もり、自分で判断する」機会を奪う。時間感覚はAIに依存するほど衰える。

| | Motion / Reclaim | Taskel |
|---|---|---|
| AIの役割 | 全自動最適化 | 鏡（自己認識の補助） |
| 時間感覚 | AIに依存（磨かれない） | 自分で校正（磨かれる） |
| 目標との接続 | ❌ なし | ✅ 双方向接続 |
| 哲学 | 効率の最大化 | 実行するたびに自分も磨かれる |

### TaskChute系（TaskChute Cloud / たすくま）

**本質:** 全時間を記録して時間感覚を自覚する

Taskelメソッドの実行レイヤーは、タスクシュートの思想を受け継いでいます。最も近いDNAを持つ存在です。

しかし、タスクシュートには「目標との階層接続」がない。「なぜそれをやるのか」の構造がなく、目標管理は別のツールに委ねられています。

| | TaskChute系 | Taskel |
|---|---|---|
| 時間順にタスクを並べる | ✅ | ✅ |
| 見積もり vs 実績 | ✅ | ✅ |
| 目標との階層接続 | ❌ | ✅ 年→月→週→今日 |
| 目標の双方向見直し | ❌ | ✅ 実行が目標を磨く |
| 心理的安全 | △ | ✅ Chute + 柔軟な判断 |
| 行動選択の最適化 | ❌ | ✅ スコアリング + Unscheduled |

### Any Planner

**本質:** ADHD向けデイリープランナー

シンプルで使いやすい。ADHD当事者への共感がある。しかし目標との接続がなく、「なぜそれをやるのか」の構造がない。

| | Any Planner | Taskel |
|---|---|---|
| 目標との接続 | ❌ | ✅ 双方向接続 |
| 時間の可視化 | △ タイムライン | ✅ 終了予定時刻 + セクション |
| AIの活用 | ❌ | ✅ 鏡として活用 |
| メソッドの提供 | ❌ | ✅ 8ステップの循環 |

## こんな方にTaskelは向いています

- 仕事ではOKRやKPIで目標を回しているのに、自分の人生にはTodoリストしか使っていない人
- フリーランスや経営者で、目標はあるが日々のタスクに追われて年初の目標が消えている人
- Todoistを3回入れ直した経験がある人
- 「今日やること」と「本当にやりたいこと」が断絶している人
- 見積もりが苦手で、いつも「時間が足りない」と感じている人
- タスク管理ツールのOverdueの赤字に疲れた人

---

## 体験の要約

> **迷わない。焦らない。責めない。やるたびに、磨かれる。**

これがTaskelの体験です。
