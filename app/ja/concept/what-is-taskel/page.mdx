
# Taskelとは？

> **Taskelは「タスク管理ツール」ではありません。年間目標と今日の行動を双方向で接続し、実行するたびに目標も自分も磨かれていく環境です。**

![Daily View](/images/daily-view.png)

---

## 7つの柱（Seven Pillars）

Taskelの体験を支える7つの設計思想です。

### 1. 目標の双方向接続

年間目標 → 月次 → 週次 → 今日のアクション。この階層が**一本の線**で繋がっています。

しかし一方通行ではありません。**今日の実行や気づきが、週次・月次・年間目標を書き換える**。直近の仕事やイベントで優先順位が変わったとき、下から上へ目標がブラッシュアップされます。

> 目標管理と時間実行を1つのツールで双方向に接続しているプロダクトは、現時点で市場にほぼ存在しません。

### 2. 時間の現実の可視化

終了予定時刻のリアルタイム表示。朝タスクを並べた時点で「今日の終了予定：23:40」と出れば、**「入りきらない」と5秒で判断**できます。

タスクを「リスト」ではなく「時間幅」で捉える。1日は24時間。使える時間はさらに少ない。この制約が「やらないことを決める力」をくれます。

![Section Management](/images/settings-schedule.png)

### 3. 実行の駆動

- **ワンクリックタイマー** — 再生ボタンを押すだけで計測開始
- **セクション** — 1日を時間帯ブロック（Morning / Afternoon / Evening等）に分割し、「次の2時間で何をやるか」に集中できる
- **集中モード（Focus）** — 実行中はタスク名と経過時間だけに集中

Taskelは計画のツールではなく、**実行を駆動するエンジン**です。

![Task In Progress](/images/task-in-progress.png)

### 4. 自己校正（Time Mirror）

見積もり時間 vs 実績時間のフィードバックループ。自分で見積もり、自分でズレを自覚し、時間感覚を磨きます。

**AIに見積もりを任せない**。これはMotion/Reclaimと真逆の哲学です。代行させたら時間感覚を磨く機会を失います。AIは「鏡」として、ユーザーの自己認識を助ける存在に徹します。

![AI Calibration](/images/ai-calibration.png)

### 5. 心理的安全

- **Chute（先送り・持ち越し）** — 今日終わらなかったタスクは、裁かれることなく翌日に流せます。「もうやらない」と判断して消すこともできます。「Overdue」の赤字地獄がありません。
- **ユーザーが自分で判断する柔軟性** — 先送りするか、やめるか、そのままにするか。システムが勝手に裁かず、ユーザーが自分で選べます。

> **未完了 = 罪にしない設計。** これがタスク沼からの脱出口です。

### 6. 習慣の自動化

ルーチン機能により、毎日・毎週の繰り返しタスクが自動で並びます。「やるかやらないか」を毎日考えなくていい。**決断疲れの排除**。

やらなかったルーチンは先送りするか、やめるか、自分で判断すればいい。システムが罰することはありません。

![Routines](/images/routines-list.png)

### 7. 行動選択の最適化

- **Unscheduled** — いつやるかわからないタスクの保管場所。今日のリストを汚さず、でも消えない。「忘れない。でも今日は邪魔しない」
- **スコアリング** — RICEフレームワークを参考に、重要度×緊急度でタスクをスコア化。重要度には目標貢献度が含まれ、年間目標に直結するタスクほどスコアが高くなります

![Planning View](/images/planning-view.png)

## 補足: プロジェクトフォルダ

7つの柱を横断的に支える整理機能として**プロジェクトフォルダ**があります。タスクをプロジェクトごとに分類・整理でき、プロジェクト単位で他人と共有することも可能です。

## 3つの設計哲学

### Assist, not Delegate（支援するが、代行しない）

AIは「鏡」として機能します。ユーザーの過去データを見せ、パターンを示唆する。しかし見積もりを代行せず、優先順位を勝手に変えません。自分で見積もり、自分で決め、自分で振り返る。この「自分で」のプロセスが、時間感覚と判断力を磨きます。

### 未完了は罰ではない

今日終わらなかったタスクは「失敗」ではなく「明日への引き継ぎ」。Chute（先送り）は正式な機能であり、正当な選択肢です。先送りするか、やめるか、そのままにするか — ユーザーが自分で判断する柔軟性を守ります。

### 目標は生き物である

年初に立てた目標が1年間変わらないのは不自然です。直近の仕事、出会い、発見、失敗。あらゆる日々の経験が目標を書き換えうる。Taskelは目標の変化を**歓迎**します。変わったから失敗なのではなく、変わったから成長しているのです。

## 体験の要約

7つの柱が生み出す体験を一文で：

> **迷わない。焦らない。責めない。やるたびに、磨かれる。**

| 柱 | 体験 |
|---|---|
| 目標接続 | **迷わない** — 何のためにやるかが明確 |
| 時間の現実 | **焦らない** — 入りきらないなら明日でいい |
| 実行駆動 | **止まらない** — タイマーが背中を押す |
| 自己校正 | **自分を知る** — 時間感覚が磨かれる |
| 心理的安全 | **自分を責めない** — 未完了が罰にならない |
| 習慣化 | **考えなくていい** — ルーチンが並んでいる |
| 行動選択 | **選びやすい** — スコアが教えてくれる |
